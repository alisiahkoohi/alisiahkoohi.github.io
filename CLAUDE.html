Project Overview

Personal academic website for Ali Siahkoohi (Assistant Professor, UCF), built with Quarto and hosted on GitHub Pages.

Build & Development Commands

quarto preview          # Local dev server with live reload
quarto render           # Build site to _site/
quarto publish gh-pages # Deploy to GitHub Pages

Architecture

  • Single-page site: All content lives in index.qmd (Quarto Markdown)
  • Configuration: _quarto.yml defines site metadata, navbar, footer, and theme settings
  • Theming: Dual light/dark theme using Bootstrap-based Quarto themes
    • Light: simplex + custom.scss (primary color: #0451A5)
    • Dark: cyborg + custom_dark.scss (primary color: #f7ac43, bg: #242424)
    • Global overrides in styles.css
  • Static assets: All in files/ — profile photo, CV PDF, publication PDFs, and BibTeX .bib files
  • Build output: _site/ (gitignored via .quarto/ pattern, untracked)

Content Structure in index.qmd

Sections in order: Background, Research, Teaching, Publications (Preprints → Journal → Conference → Thesis → Technical Reports), Miscellaneous.

Publications follow a consistent pattern: title link, author list, venue/status, and inline links to [pdf], [code], [slides], [poster], [bibtex].

Git & Deployment

  • master branch: source files
  • gh-pages branch: published site (managed by quarto publish)
  • BibTeX files are named by author-based convention: AuthorLastNamesYear.bib (e.g., SiahkoohiMorelBalestrieroEtAl2026b.bib) and stored in files/
