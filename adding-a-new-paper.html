Adding a New Paper

A new paper must be added in three places: the website, the CV repo, and bib files for both. The CV lives at ~/Documents/cv-repo/.

1. BibTeX file for the website (files/)

  • Create files/<KEY>.bib with plain text author names (no LaTeX formatting, no underline).
  • Key naming: LastName1LastName2_Year for 2-3 authors, LastName1LastName2LastName3EtAl_Year for 4+.
  • Use @unpublished for preprints, with note = {Preprint arXiv:XXXX.XXXXX}.

Example:

@unpublished{ThatipelliSiahkoohi_2026,
  title        = {
    Hypernetwork-based approach for grid-independent functional data clustering
  },
  author       = {
    Anirudh Thatipelli and Ali Siahkoohi
  },
  year         = 2026,
  note       = {Preprint arXiv:2602.22823},
}

2. BibTeX entry for the CV (~/Documents/cv-repo/bib.bib)

  • Same structure as above but Ali Siahkoohi’s name must be underlined: {\underline{\color{black}A. Siahkoohi}}
  • Use abbreviated first names (e.g., Anirudh Thatipelli stays full, but Ali is always A. Siahkoohi).
  • Add the entry at the top of bib.bib.

Example:

@unpublished{ThatipelliSiahkoohi_2026,
  title        = {
    Hypernetwork-based approach for grid-independent functional data clustering
  },
  author       = {
    Anirudh Thatipelli and {\underline{\color{black}A. Siahkoohi}}
  },
  year         = 2026,
  note       = {Preprint arXiv:2602.22823},
}

3. Website entry in index.qmd

  • Add under the appropriate ### subsection (Preprints, Journal papers, Conference proceedings, etc.).
  • Every line (title, authors, venue/year) must end with two trailing spaces () for Markdown line breaks.
  • Author names are plain text — do NOT underline Ali’s name on the website.
  • For preprints, the venue line is: *Preprint arXiv:XXXX.XXXXX*, YEAR.
  • For journal/conference papers, the venue line is: *Venue Name*, YEAR.
  • Links go on separate lines: [[pdf](...)], [[code](...)], [[slides](...)], [[poster](...)], [[bib](...)] — include only what’s available.
  • The bib link always points to: https://raw.githubusercontent.com/alisiahkoohi/alisiahkoohi.github.io/master/files/<KEY>.bib

Example preprint entry (note the two trailing spaces shown as ··):

**Hypernetwork-based approach for grid-independent functional data clustering**··
A. Thatipelli and A. Siahkoohi··
*Preprint arXiv:2602.22823*, 2026.··
[[pdf](https://arxiv.org/abs/2602.22823)]
[[code](https://github.com/luqigroup/hypercluster)]
[[poster](https://www.dropbox.com/scl/fi/...&raw=1)]
[[bib](https://raw.githubusercontent.com/alisiahkoohi/alisiahkoohi.github.io/master/files/ThatipelliSiahkoohi_2026.bib)]

4. CV entry in ~/Documents/cv-repo/AliSiahkoohi-CV.tex

  • Preprints go under \subsection*{In Preparation \& Under Review}.
  • Journal papers go under \subsection*{Journal Publications}.
  • Conference papers go under \subsection*{Peer-Reviewed Conference Publications}.
  • Each item uses \bibentry{KEY} and \href{URL}{\small \color{mygray}[label]} links.
  • Add new entries at the top of the relevant etaremune list.

Example:

    \item \bibentry{ThatipelliSiahkoohi_2026} \phantom{}\\
    \href{https://arxiv.org/abs/2602.22823}{\small \color{mygray}[pdf]}
    \href{https://github.com/luqigroup/hypercluster}{\small \color{mygray}[code]}
    \href{https://www.dropbox.com/scl/fi/...&raw=1}{\small \color{mygray}[poster]}
    \href{https://raw.githubusercontent.com/alisiahkoohi/alisiahkoohi.github.io/master/files/ThatipelliSiahkoohi_2026.bib}{\small \color{mygray}[bib]}

5. Compile CV and copy PDF

cd ~/Documents/cv-repo
pdflatex AliSiahkoohi-CV.tex
bibtex AliSiahkoohi-CV
pdflatex AliSiahkoohi-CV.tex
pdflatex AliSiahkoohi-CV.tex
cp AliSiahkoohi-CV.pdf ~/Documents/alisiahkoohi.github.io/files/AliSiahkoohi-CV.pdf
